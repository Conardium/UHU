# TRABAJO CON PERMISOS #

Como añadir atributos(permisos) a los ficheros

*Tipos de permiso: R(lectura), W(escritura), X(ejecucion) siempre en este orden

*** Hay dos formas de  dar permisos ***

** Forma de adicion y substracción (chmod) **

· chmod + a quien se lo das o quitas(u[propietario], g[grupo], o[otros], a[a todos]) + (+ o -) + rwx + ruta de los ficheros/directorios -> da(+) o quita(-) permisos a la persona sobre el fichero indicado
EJEMPLO: chmod g+w f1.txt
         chmod ug-r f1.txt
         chmod go+wx f2.txt
         chmod a-r f3.txt

· ls -l -> muestra lo que contiene el directorio + detalles sobre los permisos de los ficheros
EJEMPLO: ls -l
         -rw-rw-r-- ismael.dapalma060          users 
            ^              ^                     ^
         permisos     Propietario        grupo al que pertenece   


** Forma de aplicacion de permisos global (chmod pero con numeros), es necesario que me digan que permisos tiene cada uno de los grupos, sino no se podría aplicar esta forma**

· chmod + numero1 + numero2 + numero3 + rutas de ficheros -> segun el numero1 le da permisos al propietario, segun el numero2 le da permisos al grupo y segun el numero3 le da permisos a otros
EJEMPLO: chmod 640 f1.txt
         chmod 541 f2.txt


*En esta practica vamos a utilizar sobre todo el comando "find"

· wc + [opcion] + ruta de ficheros-> (cuenta la cantidad de caracteres(bytes), palabras o lineas del fichero)
· cal -> (muestra el calendario)
· date -> (muestra la fecha y hora actual)
· sort + ruta del fichero -> (ordena el fichero de menor a mayor por defecto), si no existe el fichero lo crea y pide entrada por teclado


** Vamos a ver tb Enlaces (Hard Link y Soft Link) **
· ln + [opciones] + origen + [destino]
si se pone la opcion -s el enlace sera Soft LinK o enlace simbolico, por defecto es el Hard link


· file + ruta fichero/s -> determina el tipo de fichero
· who -> muestra quien esta conectado y cuantas conexiones tiene
· ps -> muestra cuantos y cuales procesos(comandos) estan activos
· tty -> muestra el terminal en el que estamos trabajando
· id -> muestra nuestro identificador de la terminal
· kill + identificador(PID) -> envia una señal al proceso indicado por PID, lo mata
· write + usuario + mensaje -> envia un mensaje al usuario indicado
· msg -> si quieres o no recibir mensajes(write)
· grep + patron + ruta del fichero donde se va a buscar el patron -> busca el patron en el fichero indicado
